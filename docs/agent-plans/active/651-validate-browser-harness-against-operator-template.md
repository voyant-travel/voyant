## Agent Brief

**Category:** investigation
**Summary:** Validate the browser evidence harness against `templates/operator` so UI proof covers a full template app, not only the local workflows dashboard.

**Current behavior:**
The browser evidence runner has been proven against `apps/workflows-local-dashboard`. The operator template is a larger TanStack Start and Cloudflare-backed app with auth, route loaders, package UI surfaces, and same-origin API assumptions. We need a proof that the existing capture flow can start the template dev server, load a stable browser route, and produce useful desktop/mobile evidence.

**Desired behavior:**
A runner-managed operator workspace can start the Vite dev server on the issue-scoped port and capture the operator sign-in surface in desktop and mobile viewports without blocking browser issues. Evidence should include screenshots, videos, console logs, failed-request logs, and the summary file generated by the runner.

**Key interfaces:**
- `templates/operator` Vite dev server and TanStack Start route shell
- `pnpm agent:queue:capture-browser`
- Browser evidence validation in `scripts/lib/agent-runner-browser-evidence.mjs`
- UI package surfaces from `@voyantjs/ui`, `@voyantjs/admin`, and domain `*-ui` providers

**Acceptance criteria:**
- [ ] `templates/operator` can be served by the runner using `VOYANT_AGENT_DEV_SERVER_PORT`.
- [ ] Browser evidence is captured for `/sign-in` at `1440x900` and `390x844`.
- [ ] Captured browser evidence reports `0` console errors, `0` console warnings, and `0` failed requests, or documents an explicit accepted exception.
- [ ] The operator template passes the narrow verification lane used for this proof.
- [ ] The handoff references the browser artifact summary generated by the runner.

**Verification lane:**
- `pnpm --filter operator typecheck`
- `pnpm --filter operator build`
- `pnpm agent:queue:capture-browser -- --issue <issue> --viewports 1440x900,390x844 --url "$VOYANT_AGENT_DEV_SERVER_URL/sign-in" --dev-server-command "pnpm exec vite dev --host 127.0.0.1 --port \$VOYANT_AGENT_DEV_SERVER_PORT" --yes`

**Security:**
Not security-sensitive. This is local browser evidence for a template auth page. Do not add credentials, seed data, or external service calls.

**Artifacts required:**
Browser screenshots, videos, console log, failed-request log, dev-server log, and runner summary.

**Out of scope:**
- Changing production auth behavior.
- Adding mock users or bypassing authentication.
- Reworking the operator template UI beyond fixes required to make the proof deterministic.
