# Catalog plane migrations

Standalone drizzle migrations for the catalog plane's tables. Generated by
`pnpm -F @voyant-travel/catalog db:generate` from `src/schema.ts`.

## When to use these

There are two ways to apply catalog plane DDL to a deployment:

**Option 1 — embed in your template's migration pipeline (recommended).**

Add `packages/catalog/src/schema.ts` to your template's `drizzle.config.ts`
schema list. Your template's existing `db:generate` / `db:migrate` flow will
include the catalog tables alongside its other modules. See
[`starters/operator/drizzle.config.ts`](../../../starters/operator/drizzle.config.ts)
for the reference setup.

**Option 2 — apply these standalone migrations.**

Useful for adding the catalog plane to an existing deployment whose template
already has a long migration history. Apply the SQL files in this directory
in numeric order against your Postgres database before any vertical adopts
the catalog contract at runtime.

Both options produce the same DDL — `catalog_overlay` and
`booking_catalog_snapshot` tables with the indexes documented in the
schema files.

## Files

- `0000_catalog_initial.sql` — initial migration: creates both tables, the
  composite-unique partial index on active overlay rows, the entity lookup
  index, and the origin lookup index.

## Regenerating

If you change `src/overlay/schema.ts` or `src/snapshot/schema.ts`, regenerate
with:

```bash
pnpm -F @voyant-travel/catalog db:generate --name <descriptive_name>
```

The standalone config (`packages/catalog/drizzle.config.ts`) only sees the
catalog tables — it will never accidentally include other modules' DDL.
